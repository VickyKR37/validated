If you are seeing this page after deploying your "Validated" app, it means Firebase Hosting is NOT serving your Next.js application correctly. Instead, it's serving this placeholder HTML file.

This typically indicates an issue with the deployment or build process for your Next.js app when using the frameworksBackend feature. Common reasons include:

  • Build Failure: The Next.js application might have failed to build during the firebase deploy command. Check your terminal's deployment logs carefully for any errors.
  • Firebase Project Plan: The Next.js frameworksBackend integration relies on Cloud Functions or Cloud Run. Your Firebase project MUST be on the Blaze (pay-as-you-go) plan. The free Spark plan has limitations that often prevent these backends from deploying or running correctly.
  • Firebase CLI Version: Ensure you have a recent version of the Firebase CLI. Update via npm install -g firebase-tools.
  • Region Support: Ensure the region specified in your firebase.json (e.g., us-central1) supports Cloud Functions (Gen 2) or Cloud Run if that's what Next.js is deploying.

Recommended Next Steps:

  1. Carefully review the full output of your firebase deploy command in your terminal for any errors or warnings, especially related to building the Next.js app or deploying functions/services.
  2. Verify your Firebase project is on the Blaze plan. This is a common requirement for framework-aware hosting.
  3. Try deploying with more detailed logs: firebase deploy --debug. This might reveal more specific issues.
